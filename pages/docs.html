Installation

  1. Download VoxelGuest.jar and place it into your plugins folder.
  2. Restart/reload your server twice. Module configuration files will not be generated upon first restart/reload.

Generated Files

  • persistence.db - Stores all module data. More information is detailed below.
  • AsshatModule.properties - Contains settings that influence the behaviour of the Asshat Module.
  • GeneralModule.properties - Contains settings that influence the behaviour of the General Module.
  • GreylistModule.properties - Contains settings that influence the behaviour of the Greylist Module.

Persistence

Migration

Migration from VoxelGuest 4.x

If you are using VoxelGuest 4.x you will not be able to import most of your data yet. However, we are working on more import functionality.

  1. Delete the old VoxelGuest jar file from your plugins directory.
  2. Download VoxelGuest.jar and place it into your plugins folder.
  3. Restart/reload your server twice. Module configuration files will not be generated upon first restart/reload.
  4. Use the /vgimport (VoxelGuest Administration System) command to import your old data.

Migration from VoxelGuest 5.0 Development Builds

If you are using a development build of VoxelGuest 5.0 you need to manually update your database file or delete it (all persistent data like bans, mutes, regions will be lost) and let the plugin generate a new one. You also need to update some configuration file names. Please note, upgrading from a development build to a release build is not officially supported. The steps below might be outdated. Furthermore, these steps are not meant to as a step-by-step or click-this-button-and-then-that-one tutorials. Especially the manual database upgrading part requires a bit database/SQL knowledge.

  1. Shut down(!!!) the server.
  2. Back up your files
  3. Delete the old VoxelGuest jar file from your plugins directory.
  4. Download VoxelGuest.jar and place it into your plugins folder.
  5. Rename the following files
    1. asshat.properties to AsshatModule.properties
    2. general.properties to GeneralModule.properties
    3. greylistmodule.properties to GreylistModule.properties
  6. Make sure the strings in your config files are enclosed with " characters (check default config files below)
  7. Manually update your database file (only required if really want to keep your persistent data)
    1. Get a program to read, modify and write SQlite databases.
    2. Open persistence.db and make sure the schema matches the one found in the database documentation section below.
  8. Start your server.

Asshat Module

The Asshat Module allows server staff to manage and restrict the extent to which a player may interact with the server. Banning, kicking, muting, freezing, and recalling ban data are all possible through this module.

Commands and Permissions

ban

Bans the player specified in the playername argument from the server for the reason specified in the reason argument. Banned users may not reconnect to the server. If the specified player is online, player detection is enabled and the entire name need not be typed.
  • vban
  • voxelguest.asshat.ban
Argument Required Description Example
playername yes The player to ban /ban MrBoobs
reason no The reason you are banning the player for /ban MrBoobs He sucks
-si no Suppresses ban broadcast /ban MrBoobs -si
-f no Bans an offline player (name has to be spelt correctly) /ban MrBoobs -f

unban

Unbans the player specified in the playername argument from the server.
  • vunban
  • voxelguest.asshat.unban
Argument Required Description Example
playername yes The player to unban /unban MrBoobs
-si no Suppresses unban broadcast /unban MrBoobs -si
-f no Bans an offline player (name has to be spelt correctly) /unban MrBoobs -f

banreason

Returns the reason for which the player specified in the playername argument has been banned. Names of offline players must be spelt completely with all numbers, letters, and symbols, but capitalization is not important.
  • vbanreason
  • voxelguest.asshat.banreason
Argument Required Description Example
playername yes The player you want to get the banreason of /banreason MrBoobs

kick

Kicks the player specified in the playername argument from the server.
  • vkick
  • voxelguest.asshat.kick
Argument Required Description Example
playername yes The player to kick /kick MrBoobs
reason no The reason you are kicking the player for /kick MrBoobs He sucks
-si no Suppresses kick broadcast /kick MrBoobs -si

mute

Denies the player specified in the playername argument the ability to speak in chat.
  • vmute
  • gag
  • vgag
  • voxelguest.asshat.mute
Argument Required Description Example
playername yes The player to mute /mute MrBoobs
reason no The reason you are muting the player for /mute MrBoobs He sucks
-si no Suppresses mute broadcast /mute MrBoobs -si

unmute

Re-allows the player specified in the playername argument the ability to speak in chat.
  • vunmute
  • ungag
  • vungag
  • voxelguest.asshat.unmute
Argument Required Description Example
playername yes The player to unmute /unmute MrBoobs
-si no Suppresses unmute broadcast /unmute MrBoobs -si

soapbox

Denies all players the ability to speak in chat. Resend the command to re-allow all players the ability to speak in chat.
  • vsoapbox
  • voxelguest.asshat.soapbox (Allows command usage.)
  • voxelguest.asshat.bypass.silence (Prevents a player from being muted by soapbox.)
This command doesn't have any arguments.

freeze

Denies all players the ability to move. Resend the command to re-allow all players the ability to move.
  • vfreeze
  • voxelguest.asshat.freeze (Allows command usage.)
  • voxelguest.asshat.bypass.freeze (Prevents a player from being movement impaired by freeze.)
This command doesn't have any arguments.

General Module

The General Module contains useful commands for both staff and users of all usergroups. Users may view an organized list online players, track server TPS, and alert other users that they are going AFK. Staff may teleport to users, change usergroups, vanish, fakequit, and view system data.

Commands and Permissions

who

Lists all online players, organized by usergroup. Players flagged as AFK display a light gray name in the player list.
  • vwho
  • list
  • voxelguest.general.who
This command doesn't have any arguments.

vanish

Hides you character model and nameplate from players without vanish permission.
  • vvanish
  • van
  • voxelguest.general.vanish
This command doesn't have any arguments.

fakequit

Hides you character model and nameplate from players without vanish permission. Additionally hides your name in the online player list from all players without fakequit permissions. A fake disconnect message is displayed on behalf of the player fakequitting.
  • fq
  • vfq
  • voxelguest.general.fakequit
This command doesn't have any arguments.

ep

Removes all entities except paintings and item frames in the specified world. The world argument can be replaced with an asterisk (*) to target all worlds.
  • vep
  • voxelguest.general.ep
Argument Required Description Example
world yes The world to remove entities from /ep MyCoolWorld

sys

Returns information regarding CPU performance and memory usage amongst other data.
  • system
  • vsys
  • voxelguest.general.sys
Argument Required Description Example
mem no Prints more detailed memory usage information /sys mem
gc no Triggers Java's native garbage collector /sys gc
lag no Prints the server's TPS (ticks per second) only /sys lag

vpg

Assign a user group to a player.
  • vpromote
  • voxelguest.general.vpg
Argument Required Description Example
playername yes The player to assign to a group /vpg MrBoobs guests
usergroup yes The group to add a player to /vpg MrBoobs guests

vtp

Teleports a player.
  • vteleport
  • voxelguest.general.vtp
Argument Required Description Example
playername no The player to teleport to OR the player to teleport - see arguments below /vtp MrBoobs teleports you to MrBoobs
me no, requires playername Teleport player specified in playername to your location /vtp MrBoobs me teleports MrBoobs to you
x# y# z# no Modify destination coordinates relative to the target player /vtp MrBoobs x5 x10 z5 teleports you to MrBoobs with an offset of (5|10|5)

afk

Flags your account as AFK and displays a message indicating the action to all players on the server.
  • vafk
  • voxelguest.general.afk
Argument Required Description Example
reason no Sets the reason you've gone AFK for. A random AFK message will be selected if this argument is omitted. /afk School

addafkmessage

Adds an AFK message that may be selected at random when a player goes AFK.
  • addafkmsg
  • vaddafkmsg
  • voxelguest.general.createafkmsg
Argument Required Description Example
text yes The message to add to the list of random messages /addafkmessage Pointless message

watchtps

Rather than displaying your current experience level, your experience bar will display the server's TPS and update in real time. Resend the command to return your experience bar to displaying your current experience level.
  • vwatchtps
  • voxelguest.general.watchtps
This command doesn't have any arguments.

vgpg

Allows to create a VoxelGuest player group that maps permission system group names to /who group colors.
This command doesn't have any aliases.
  • voxelguest.general.playergroup
Argument Required Description Example
colors no Displays all available colors /vgpg colors
group-name yes (no if 'colors' has been passed as an argument) The name of the group in your permission system /vgpg Guests green
color yes (no if 'colors' has been passed as an argument) The color to use in /who for the group specified in group-name /vgpg Guests green

Greylist Module

The Greylist Module allows a server's administration to filter direct access to the server through greylisting and whitelisting, and control demographics through a very simple and effective system.

Commands and Permissions

greylist

Greylists a player. Greylisted users may access the server at any time. This command automatically promotes the player specified in the playername argument to the usergroup defined as the gl-group-name in the greylist module configuration.
  • gl
  • voxelguest.greylist.greylist - Allows to use /greylist command
  • voxelguest.greylist.override - Allows a player to bypass greylist check on join
Argument Required Description Example
playername yes The player to greylist /greylist MrBoobs

ungreylist

Ungreylists a player.
  • ugl
  • voxelguest.greylist.ungreylist - Allows to use /ungreylist command
  • voxelguest.greylist.override - Allows a player to bypass greylist check on join
Argument Required Description Example
playername yes The player to ungreylist /ungreylist MrBoobs

whitelist

Whitelists a player. This command automatically promotes the player specified in the playername argument to the usergroup defined as the wl-group-name in the greylist module configuration.
  • wl
  • voxelguest.greylist.whitelist
Argument Required Description Example
playername yes The player to whitelist /whitelist MrBoobs

Helper Module

The Helper Module serves as a systematic method of communication between guests and helpers. Logs are maintained to keep statistical data regarding the frequency of a player's interaction with the module and helper notes regarding feedback for players that use the module's functionality.

Commands and Permissions

wlreview

Submits a ticket to helpers indicating that the triggering player would like to be reviewed for whitelisting.
  • vwlreview
  • voxelguest.helper.wloveride - Disallows a player to submit a ticket. This behaviour might be changed in the final 5.2.0 release
This command doesn't have any arguments.

helper

Adds or removes a player form the list of helpers.
  • vhelper
  • voxelguest.helper.helper
Argument Required Description Example
--add [playername] no Adds the player specified in playername to the list of helpers /helper --add MrBoobs
--remove [playername] no Removes the player specified in playername to the list of helpers /helper --remove MrBoobs

helperreview

Allows a helper to examine the helper review history of a player or comment on one instance of a player's review history.
  • hreview
  • voxelguest.helper.review
Argument Required Description Example
-c [comment] no Adds a review comment /helperreview -c Needs moar materials!
-h [playername] no Displays a players review history /helperreview -h MrBoobs

VoxelGuest Administration System

The Administration System encompasses several commands to manage VoxelGuest.

Commands and Permissions

vmodules

This command allows you to list, enabled and disable VoxelGuest modules.
  • vgmodules
  • voxelguest.manage.modules
Argument Required Description Example
--list, -l no Lists all avaiable modules /vmodules -l
--enable [module-name], -e [module-name] no Enables a module /vmodules -e RegionModule
--disable [module-name], -d [module-name] no Disables a module /vmodules -d RegionModule
Available Modules
  • AsshatModule
  • GeneralModule
  • GreylistModule
  • HelperModule
  • RegionModule

vgimport

This command allows you to list, enabled and disable VoxelGuest modules.
  • voxelguestimport
  • guestimport
  • import
  • voxelguest.manage.import
Argument Required Description Example
-bans no Imports banned players from VG 3.x and 4.x. The old file(s) have to be in the VoxelGuest folder in the plugins directory. /vgimport -bans
-greylist no Imports the greylist from VG 4.x. The old file(s) have to be in the VoxelGuest folder in the plugins directory. /vgimport -greylist
-afkmessages no Imports the random AFK messages from VG 4.x. The old file(s) have to be in the VoxelGuest folder in the plugins directory. /vgimport -afkmessages
